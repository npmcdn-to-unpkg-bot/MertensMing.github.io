Node.js 学习之路 —— NPM

发表于 2016-09-05   |   分类于 Javascript

npm 有两层含义：

  1. Node 的开放式 模块 登记和管理 系统，网址是 npmjs.org
  2. 另一层含义是 Node 默认的模块管理器，是一个命令行下的软件，用来安装和管理 Node 模块。

npm 常见的使用场景有以下几种：

  • 允许用户从 npm 服务器下载别人编写的第三方包到本地使用。
  • 允许用户从 npm 服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到 npm 服务器供别人使用。

起步

更新 NPM

npm 是不需要单独安装的，因为在安装 Node 的时候，会连带一起安装 npm，但是，Node 附带的 npm 可能不是最新的版本，最好使用以下命令跟新到最新版本。

  • @latest表示最新版本
  • -g表示全局安装

新的 package.json 文件

npm init 用来初始化生成一个新的 package.json 文件。
它会向用户提问一系列问题，如果你觉得不用修改默认配置，一路回车就可以了。

如果使用了-f（代表force）、-y（代表yes），则跳过提问阶段，直接生成一个新的 package.json 文件。

设置环境变量

npm set用来设置环境变量。

上面命令等于为npm init设置了默认值，以后执行npm init的时候，package.json的作者姓名、邮件、主页、许可证字段就会自动写入预设的值。
这些信息会存放在用户主目录的 ~/.npmrc文件，使得用户不用每个项目都输入。
如果某个项目有不同的设置，可以针对该项目运行npm config。

查看模块的具体信息

npm info 命令可以查看每个模块的具体信息。

  • 这个对象的每个成员，都可以直接从info命令查询。

列出安装的所有模块

npm list命令以树型结构列出当前项目安装的所有模块，以及它们依赖的模块。

加上global参数，会列出全局安装的模块。

npm list命令也可以列出单个模块。

安装模块

基本用法

Node 模块采用 npm install 命令安装。每个模块可以“全局安装”，也可以“本地安装”。

全局安装

“全局安装”指的是将一个模块安装到系统目录中，各个项目都可以调用。一般来说，全局安装只适用于工具模块，比如eslint和gulp。

本地安装

“本地安装”指的是将一个模块下载到当前项目的node_modules子目录，然后只有在项目目录之中，才能调用这个模块。

直接输入Github代码库地址

npm install也支持直接输入Github代码库地址。

安装之前，npm install会先检查，node_modules目录之中是否已经存在指定模块。如果存在，就不再重新安装了，即使远程仓库已经有了一个新版本，也是如此。

强制重新安装

如果你希望，一个模块不管是否安装过，npm 都要强制重新安装，可以使用-f或--force参数。

重新安装所有模块

如果你希望，所有模块都要强制重新安装，那就删除 node_modules目录，重新执行npm install。

指定参数

安装不同的版本

install命令总是安装模块的最新版本，如果要安装模块的特定版本，可以在模块名后面加上@<版本号>。

如果使用--save-exact参数，会在package.json文件指定安装模块的确切版本。

指定依赖关系

指定所安装的模块属于哪一种性质的依赖关系(生产环境的依赖还是开发环境的依赖)，即出现在packages.json文件的哪一项中。

  • –save：模块名将被添加到dependencies，可以简化为参数-S。
  • –save-dev: 模块名将被添加到devDependencies，可以简化为参数-D。

更新与卸载

npm update

npm update命令可以更新本地安装的模块。

它会先到远程仓库查询最新版本，然后查询本地版本。如果本地版本不存在，或者远程版本较新，就会安装。

使用-S或--save参数，可以在安装的时候更新package.json里面模块的版本号。

npm uninstall

npm uninstall命令，卸载已安装的模块。

执行脚本
