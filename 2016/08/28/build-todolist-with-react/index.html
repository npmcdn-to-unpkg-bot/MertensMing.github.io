开发脚手架

webpack.config.js

这里就不赘述了，可以看看 Webpack 配置 这篇博客，需要对 webpack 有一定的了解。还有 webpack.config.js 文件里面的注释也是比较详细的了。

这里需要注意的一点是，这个环境的配置都分为了两个环境，开发环境（dev） 和 生产环境（prod），所以你会看到类似于下面的文件名。

生产环境 和 开发环境的配置具体有哪些不一样下我也会提到。

index.js

这个文件是整个项目的入口，个人觉得这里需要注意的地方就是热加载（react-hot-loader），因为需要 hot-load，所以被 <Provider></Provider> 包裹的部分必须作为一个组件（<Root>）import 进来。store 和 history 也需要在这里作为属性传递给 <Root>。
